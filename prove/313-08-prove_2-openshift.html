Deploy your application to OpenShift

In order to deploy your application to OpenShift, you will need to create a new Java application gear.

You should not remove the PHP gear at this point (especially if you have not received a score for your project yet!).

To create your new gear:

  1. Login to OpenShift and create a new application
  2. Select "Tomcat 7" as the type.
  3. Choose a name, for example, "java". Leave the source code git repository blank, select "no scaling", and then click "Create Application".
  4. This may take a few minutes while your Java application server is set up and configured.

Choose your own deployment adventure

At this point, you have a choice to make about how you will build and deploy your application:

Will you let OpenShift build the Java application from your source, or will you pre-build a WAR file (Java Web ARchive) and push that up?

The most straight-forward option is to simply build a WAR file and deploy it. This is listed as option #1. In industry, this type of build process (building a .war file and deploying it) is common for deploying to staging and live servers. The deployment would usually not be done via Git (it is for source control not compiled files), but that is the way OpenShift uses to deploy files, so that is the way you do it if you choose to deploy a .war file.

The second option is to push your source to OpenShift along with a Maven build file and let OpenShift build it. This is nice because you don't have to put a compiled .war file into your Git repository (which seems a little awkward) and also, all the dependencies can be managed via your Maven file and pulled onto the server automatically. This is common in industry for deploying to test and development servers, because your Maven build can also be configured to run unit tests, etc. Because there are a few more moving pieces in this approach, there are more chances for something to go wrong. That said, once you have it set up and working, it should not be any more work to deploy.

For the purposes of this class, any of these options (or something else) is perfectly fine. The important thing is that you are able to deploy your application to the Web.

  2. Let OpenShift build your application
