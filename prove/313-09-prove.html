JSP DISCUSSION THREAD

OBJECTIVE

The objective for this assignment is to become more familiar with JSPs and Servlets.

OVERVIEW

Create a simple discussion forum where a user must login and then can post comments to a growing list. Store your usernames and passwords in a file on the server and store the posts in another file. Posts should be displayed in reverse chronological order (i.e., most recent at the top). You do not need to worry about creating a “sign up” page or anything of that nature.

INSTRUCTIONS

You should have the following pages:

  • Sign in
  • Invalid Login
  • Enter new post
  • View posts

The interaction should be as follows:

  • From the “Sign in” page
    • Successful login: Redirect to new post page
    • Unsuccessful login: Redirect to invalid login page
  • From the “Invalid login” page
    • Display a message about the failed login
    • Have a link back to the sign in page
  • From the “Enter new post page”
    • Have a link to go directly to the view posts page (without entering a post)
    • Have a text area with and a submit button. Upon submission, redirect to the view posts page.
  • From the “View posts” page
    • Display all the posts that have been submitted in reverse order.
    • Posts should display the username that generated them, the time created, and the content of the post.
    • Have a link to go back to the enter new post page.

To handle this interaction, you should have at least two servlets:

  1. A “Sign in” servlet. This servlet should validate the username and password and then transfer to either the new post page, or the invalid login page.
  2. A “Create post” servlet. This servlet should accept a new post, save it to the file and then transfer to the view posts page.
  3. You may additionally want to have a “Load posts” servlet that would load the posts from the file, and save them to memory, and then transfer control to the“View Posts” JSP to display them. In other words, you should use these servlets as your controllers, and have the JSP’s simply function as views, in the MVC pattern.

FILE FORMAT

You may format the files however you like.

SUBMISSION
