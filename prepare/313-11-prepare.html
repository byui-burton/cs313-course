PREPARATION

Before meeting with your team, please review the following material on database access using Java. Please note that in order to interact with a database, you need to have a MySQL instance that you can interact with. Thus, while you are developing your code you need to do one of the following:

  1. Deploy your development code to your server where your MySQL instance exists.
  2. Install a local instance of MySQL for development
  3. Open the MySQL port on your server to allow remote connections, so that you can connect directly to it from your local machine.

Option 1 (deploying your code for each test), can be cumbersome, especially if you are not using an automated build/deployment process.

Option 3 can be a major security risk. It is best practice to NOT allow direct connection to your database from outside your server(or data center/VPN, if your application is not all running on the same server).

Thus, the best approach is to install MySQL locally on your development machine, so you can test and debug code there, and then when it is ready to deploy, you actually deploy it to your server and do final testing there.

As noted in the tutorial, you will need to get the JDBC driver for your database (in our case MySQL). This can be downloaded directly from this MySQL.com page. You will download a zip file containing source and documentation as well as a JAR file. This JAR should be put into your lib directory. (Please note that after selecting the platform and the download type it will ask you to sign in or create an account, but this is NOT necessary. There is another link below that allows you to simply download the driver). If you download the “platform independent” version, it will be a zip file that contains the necessary JAR along with some other files. If you download the windows installer, running it will place the necessary files in a directory similar to C:\Program Files(x86)\MySQL.

Alternatively, if you are using Maven, you can simply add this dependency to your pom.xml file (instead of manually downloading the JAR file):

<dependency>
  <groupId>mysql</groupId>
  <artifactId>mysql-connector-java</artifactId>
  <version>5.1.34</version>
</dependency>
