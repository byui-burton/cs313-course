Authentication: The goal here is not to become a security expert, but to at least be aware of some of the common vulnerabilities and techniques.
  • Perhaps the best way to handle authentication on your PHP site is to use an external library/service to handle it. There are many intricacies associated with security, so entrusting these issues to someone who has gone through the details rather than reinventing the wheel can be a really good idea. One such approach is to use OpenID.
  • If, however, you are going to store usernames and passwords in your application, it is critical (and easy!) to at least hash the passwords rather than storing them directly. See Password Hashing  and Safe Password Hashing.
  • Also consider these Best Practices for Web App Authentication.

AJAX and PHP: For the most part, AJAX should be a review from previous classes in client side development.

    • Using a client side library.
      • While AJAX can be done using basic JavaScript, many well-established libraries exist that can make the job much easier. In industry it is very standard to use these libraries for client side interaction.
      • jQuery  is one good option to consider as a lightweight and easy to use library (there are obviously many others as well).
    • JSON
      • While your PHP page can return any data back for your JavaScript to parse, it is often helpful to return JSON. PHP has a built-in function to encode variables/arrays in JSON format: json_encode.
      • If you are returning JSON, you should set the MIME type to match. Putting this at the top of your PHP page will do the trick:
        • header(‘Content-type: application/json’);
