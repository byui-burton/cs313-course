SETTING UP GITHUB

OVERVIEW

In this course we assume you already have basic experience with Git from your earlier courses. If you do not, you will still be ok, but you will need to come up to speed on your own. Here is a tutorial that gives a good introduction to the subject: https://www.youtube.com/watch?v=0fKg7e37bQE and here is a link to some good documentation: http://git-scm.com/book/en/v2/Getting-Started-Git-Basics .

In order to share our code with one another (so we can get ideas from each other) and also, to make the code available to the instructor for feedback, we also want to push our code to a public repository. (Also, later in the semester, we will be working on team projects and this will facilitate sharing code with teammates.)

Thus, we are going to set up a public repository at GitHub and configure our server to automatically pull and deploy changes when they are committed to GitHub.

INSTRUCTIONS

The following steps will help you set up a GitHub repository and push a simple Hello World page to it. (These assume you are using the command line for Git; you are welcome to use a GUI if you would like, but it will not be discussed here.):

  1. Go to GitHub.com and sign up for a new account if you haven't already.
  2. On GitHub, create a new repository, and choose a name for it (e.g., cs313php).
  3. After creating the repository, it should give you a URL to use, for example: https://github.com/sburton42/cs313php.git
  4. On the command line (GitBash on Windows), change to parent directory of where you want project to be.
  5. Clone your new (likely empty) repository, e.g. "git clone https://github.com/sburton42/cs313php.git" (substituting your own username and repository name of course)
  6. This should create a subdirectory with the same name as your repository. Change into that directory.
  7. Create a new index.html page in this directory with your editor of choice. This is a simple page that just needs to display "Hello World".
  8. Now add your page to your local git repository, commit it, and push it to GitHub. For example:
    1. git add index.html
    2. git commit -m "adding hello world page"
    3. git push origin master
  9. Verify that your code is at GitHub, by browsing to the repository on GitHub.com.
The next step is to set up your OpenShift server and connect it to GitHub. See: Setting Up OpenShift
