Setting Up Github

Before starting these instructions, make sure you have Set up your Heroku application and used Git to deploy content to the cloud.

Overview

In order to share our code with one another (so you can get ideas from each other) and also, to make the code available to the instructor for feedback, you'll also need to push our code to a public repository. (Also, later in the semester, you will be working on team projects and this will facilitate sharing code with teammates.)

Thus, in addition to having a Git repository on your local computer and one at Heroku that you deploy to, you will also set one up at GitHub and push your content to it as well. We we add GitHub as a second "remote" server for our local repository so that after committing our code, we will have the ability to push it to either Heroku or GitHub.

There are ways that Git can be configured to automatically push to both remotes, but a more simplistic way is to have two separate remotes, and then you have the freedom to push to one or the other whenever you like.

Instructions

  1. Go to GitHub.com and sign up for a new account if you have not done so already. If you have an account from past projects or classes, it will work great for this course.
  2. At GitHub, create a new repository, and choose a name for it (e.g., cs313-php). Leave it public, and don't initialize it with anything (even a readme or .gitignore)--it should be left completely blank.
  3. After creating the repository, it should give you a URL to use, for example: https://github.com/sburton42/cs313-php.git
  4. On the command line (GitBash on Windows), change to the directory of the repository that you created to push your content to Heroku.
  5. Type "git remote -v" to list all the available remotes. At this point you should only see your Heroku remote listed and possibly the one you initially cloned from.
  6. Use "git remote add" to add another remote. You first give an alias (e.g. github) and then the URL. For example (substituting your own GitHub URL of course):
    git remote add github https://github.com/sburton42/cs313-php.git
  7. Now type, "git remote -v" again and this time you should see the heroku remote and the GitHub one.
  8. Push your latest changes to GitHub, by specifying the alias, "git push github master".
  9. Verify that your code is at GitHub, by browsing to the repository on GitHub.com (refresh the page if necessary).
  10. Now that you have set up both remotes, you can push to either one, by specifying the alias, for example "git push github master" or "git push heroku master".
