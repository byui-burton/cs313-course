Each group member should read the preparation material before meeting with the team to ensure that the team meeting can be as efficient as possible.

Please note that the password_hash functions are available in PHP 5.5, but our OpenShift servers are only using PHP 5.4. There is a forward compatibility library available at: https://github.com/ircmaxell/password_compat/blob/master/lib/password.php

Download the contents of this php file, save it as a file "password.php" in your current directory, and then you can require and use this file just the same is if you were using PHP 5.5.

TEACH ONE ANOTHER

Your assignment is to create a simple login page and allow users to sign up with a new username and password.

INSTRUCTIONS

1. Create a simple database table for users with an ID, a username, and a password (to store hashed passwords).

2. Create three pages (described in more detail below), a homepage, a sign-in page, and a sign-up page. You may wish to create additional PHP pages to help in the processing of the submitted values.

3. Create a sign-up page that asks for a username and password, and then inserts this into the database.

  • Don’t insert the password directly into the database, use the password_hash() function to generate a hash of it, and insert that into the database.
  • After inserting the user, redirect to the sign-in page.

For redirects, you can specify the new location in the header.

 header(‘Location: ’.$newURL);

4. Create a sign-in page that prompts for a username and a password. (Make sure to use a password HTML element, rather than a text box).

  • If a correct username/password is entered, save the user id to the session and redirect to the homepage.
  • If an incorrect username/password is entered, stay on this page.
  • Make sure to use the password_verify() function and compare against the hash in your database.
  • This page should have a link to the sign-up page.

5. Create a “homepage” that checks for a current “logged in” user.

  • If a user is found, display, “Welcome [username]” (where [username] is the current user’s username.)
  • If no user is logged in, redirect to the sign-in page.

PROVE

For this assignment, you will likely produce two to three PHP files. Each member should submit each of these to I-Learn. You need not submit your database.

As a stretch challenge, you may complete the following steps for extra credit:

Stretch Challenge 1: Add two password boxes to the sign-up page, and ensure that they match before inserting. If they don’t, stay on that page, but display a message in red, and put red asterisks next to the boxes.

Stretch Challenge 2: Add to stretch challenge 1 the requirement that a password must be at least 7 characters and contain a number.

Stretch Challenge 3: Add a client-side check for stretch challenges 1 and 2 to detect these conditions without requiring a page submission.

Please indicate in your submission comments if you completed the basic task, and which stretch goals you completed.
