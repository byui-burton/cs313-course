TEACH ONE ANOTHER

OBJECTIVE

Design a simple relational database structure and create the schema in a MySQL database.

INSTRUCTIONS

Consider that you have been asked to write an application to store notes for users for general conference talks. The basic requirements are:

  1. This system must track notes for multiple users.
  2. The speakers should be tracked separately from the conference because one speaker will speak at multiple conferences.
  3. Individual sessions of conference (e.g., Saturday morning, Sunday afternoon) need not be tracked, rather simply the year and month, (e.g., April 2009, Oct 2012).
  4. You need not account for the fact that some speakers will speak multiple times at a conference, but assume that a note can be tied to a “speaker + conference.”
  5. Multiple notes can be added by the same user about a single conference talk.

Your assignment is to create the database for this application. Please take into account good database design principles such as the elimination of redundant data, data type selection, primary keys, and foreign key relationships.

The first step is to put together a design for the database (i.e., outside of your MySQL environment). You may want to use a collaborative image drawing or document editing application for this. Once you have determined the tables you will need, the data that will go into each one, and the relationships between them, you should then log into MySQL and create these tables in a test database.

For this assignment, it is not required to populate these tables with data, only to implement the schema (i.e., create the tables and their relations).

PROVE

To demonstrate your tables, please list each table, show its details, and show the create statements for each. This can be accomplished with the following commands in MySQL:

SHOW TABLES; (this will list all the tables in your database)

Then for each table:

DESCRIBE table_name; (this will list the columns for the table “table_name”, fill in your name here

SHOW CREATE TABLE table_name; (this will show the create statement for the table “table_name”)

After running these commands copy and paste the output into a text document and submit to I-Learn.

Each student should submit a copy of the assignment to I-Learn. Please note that these should be identical, but it is important to make sure that each student on the team has a copy of the work.
