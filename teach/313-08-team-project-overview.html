Java Project Overview

Overview

For the second half of the semester we will be building web applications using Java. As with our PHP project in the first half of the semester, this will be an extended project that will drive the assignments and much of our learning for the rest of the semester.

Basic Guidelines

One of the strengths of using Java and separating the application into tiers is the ability to handle more complicated logic, rules, and interaction of objects. To help facilitate an application with more complex objects and interaction in a shorter amount of development time, we will be using objects from third party APIs.

In this regard the basic guidelines for the project are:

  • Build something cool
  • Create a Web-based Java application
  • Use an API (calling it from your Java code), for example:
    • FamilySearch
    • Facebook
    • Twitter
    • Amazon
    • Instagram
    • Twilio
    • YouTube
    • Pinterest
    • BigOven
    • Etc.
  • Interact with a database (optional) - If you are only lightly using an API above, then you should also plan to have a database component. If you are using an API (or multiple APIs) extensively and it doesn't make sense to include a database in your project, you may omit this.
  • Separate your application into three tiers (User Interface, Business Logic, Data Access)
